A Minimal Spark RDD Setup in IntelliJ 4/21/22

Hello, blog! It's been about a year since my first post so I thought it was time I make another contribution here :). My (last!) spring quarter at Cal Poly is underway, but I'm still reflecting on the things I learned last term. One thing in particular stands out as interesting which was using Scala and Apache Spark RDDs for distributed computing problems. RDDs, unlike MapReduce, are actually cool. To me it feels like the intersection of sleek higher-order functions and big data processing. I took a data mining class last quarter and let's just say that using Spark RDDs instead of lengthy python/pandas scripts could have saved me and my classmates both scraps of dev time and literal hours spent letting computations run (we were working with some large datasets). Another key advantage of Spark over another distributed model like MapReduce is that writing semi-complex queries can be pretty quick, for example, writing a K-Nearest-Neighbors classification algorithm in one (mostly) readable line is surprisingly straightforward, especially once you're familiar with some Spark tricks. Anyways, the real reason for writing this post is not just to talk about why Spark but more importantly, _how_ Spark. There's a lot of environments you could use but since I'm using Scala, IntelliJ was probably my best option. However, when looking for a tutorial on the web to Wiki-How me into RDD greatness, I was surprised I couldn't find any guides simple enough for me to get things running, so that's exactly what I hope to do; for any future students that run into this issue and somehow find my blog despite 0 SEO :). Here were my steps:

  1. Download IntelliJ
    2. You can get the community version here
  2. Install the Scala plugin
    3. There's a plugins tab on the side of the IntelliJ startup window. That's where you'll go to download the Scala plugin!
  3. Create a new project
    4. From the projects tab on the startup window, click “New project” and you should end up at a menu that looks like this.

    Choose sbt as your project type.
  4. Choose your project specs
    5. This is where I first ran into some issues. For some reason, I've only been able to get this to work with my sbt version set to 1.0.4, Java 8, and Scala version 2.11.8

  5. Build your project
    6. When you click “finish”, your project should get created, be sure to give it a few seconds to extract the structure from the project specs and then you should be set.
  6. Get your build.sbt file right
    7. The build.sbt file is where you include Spark from Apache and set your version, along with a couple other things. Here's the template that worked for me:

    Once you have your sbt file setup, right click on it in the file navigator and select “build module *your app name*”, this process should only take a second.
  7. Go crazy!
    8. You should be all set to start using RDDs! Create a scala object (in src/main/scala) and use the following import:
    import org.apache.spark.{ SparkConf, SparkContext }

Init. 3/25/21

Hello. This is my first blog post! I know I'm about twenty years late to the blogging scene but if the IndieWeb movement gains any serious traction over the next few years (unfortunately, it won't) then I might just be a few years ahead of it's big resurgence. I'm actually half kidding, I think blogging is thriving on certain parts of the internet! I still like to read through a handful of personal blogs including those by Cal Newport , Anton Krupicka , and my friend Maddie Pringle .

Regardless of the current state of blogging and the web, I've decided this has the potential to be an enjoyable way for me to record some thoughts, develop a creative outlet, and express my agency and individuality in a way that is independent of the engagement based validation loops of modern social platforms. I'm not really sure what exactly I want to do with this blog yet. Maybe I'll write about running or other outdoor endeavors, college, books, or whatever is interesting to me at the time. Maybe someone will find it interesting, or maybe they won't. I guess that could be the simple beauty of a blog, freedom to write about whatever, for my own sake, and on my own terms.

I don't feel as though I have anything particularly interesting or important to contribute to the infinite knowledge base that is the internet, but I think that's okay. I'm sure I'll continue to wrestle with the question of whether or not blogging is a worthwhile activity but for now, I think it will be fun! So let's see where this goes!
